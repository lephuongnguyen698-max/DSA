BÀI 4: PHÂN TÍCH ĐỘ PHỨC TẠP CỦA THUẬT TOÁN TÌM KIẾM TUYẾN TÍNH

1. Phân tích số phép so sánh (với mảng có n phần tử)

(a) Trường hợp tốt nhất:
- Xảy ra khi: Phần tử cần tìm x nằm ngay đầu mảng (vị trí chỉ số i = 0).
- Số phép so sánh: 1 phép so sánh.

(b) Trường hợp xấu nhất:
- Xảy ra khi: Phần tử cần tìm x nằm ở cuối mảng (vị trí n-1) hoặc không có trong mảng.
- Số phép so sánh: n phép so sánh.

(c) Trường hợp trung bình (khi phần tử chắc chắn có trong mảng):
- Xảy ra khi: Phần tử x có xác suất xuất hiện đồng đều tại bất kỳ vị trí nào từ 1 đến n.
- Công thức tính: (1 + 2 + 3 + ... + n) / n = [n * (n + 1) / 2] / n = (n + 1) / 2.
- Số phép so sánh: (n + 1) / 2 phép so sánh.

2. Độ phức tạp thời gian theo ký hiệu O lớn (Big O)

Từ các phân tích số phép so sánh ở trên, ta suy ra độ phức tạp thời gian như sau:

- Trường hợp tốt nhất: O(1)
- Trường hợp xấu nhất: O(n)
- Trường hợp trung bình: O(n)

Kết luận: 
Độ phức tạp tổng quát của thuật toán tìm kiếm tuyến tính là O(n). Thời gian chạy của thuật toán tỷ lệ thuận với số lượng phần tử của mảng.
